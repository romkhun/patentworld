## Issues Found

### 1. Heading hierarchy skip (1 page)
- **`/chapters/inv-team-size`** (`src/app/chapters/inv-team-size/page.tsx`, line 485): Uses `<h4>` without a preceding `<h3>`, creating a skip from h2 to h4. This violates heading hierarchy best practices for accessibility and SEO.

### 2. Missing OpenGraph / Twitter / canonical on deep-dive-overview (1 page)
- **`/chapters/deep-dive-overview`** (`src/app/chapters/deep-dive-overview/layout.tsx`): Has a basic `Metadata` export with title and description only. Missing OpenGraph, Twitter Card, canonical URL, and JSON-LD structured data. All other chapter pages have these via the `chapterMetadata()` and `chapterJsonLd()` helpers.

### 3. deep-dive-overview not in CHAPTERS array
- The `deep-dive-overview` page exists at `/chapters/deep-dive-overview` but is not registered in the `CHAPTERS` array in `src/lib/constants.ts`. This means:
  - It is excluded from the XML sitemap generated by `src/app/sitemap.ts`
  - It cannot use the `chapterMetadata()` / `chapterJsonLd()` helpers (which look up by slug in CHAPTERS)
  - It has no chapter number assigned

### 4. HERO_STATS visualization count (minor)
- `HERO_STATS.visualizations` is set to **458**, which matches the chapter-page ChartContainer total. Including the homepage's own ChartContainer, the actual site-wide count is **459**. This is intentional (the homepage chart is a decorative preview, not a standalone analytical visualization), so this is informational only.

### 5. No user-facing sitemap HTML page
- There is no `/sitemap` HTML page for users. Only `sitemap.xml` is generated programmatically. This is standard practice and not necessarily an issue.
